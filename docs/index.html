3D Index

Compute I(m, e) q-series for any cusped manifold in the SnapPy census.

Refined Index

Two-variable (q, t) refinement with automatic basis selection.

Dehn Filling

Fill cusps with (P, Q) slopes and compute the filled index.

Non-closable Cycles

Search for slopes whose filled index vanishes identically.

Data Packs

Download pre-computed kernels and census indices to skip expensive computations.

Installation

macOS

The app is ad-hoc signed but not notarized with Apple, so macOS will block it on first launch. Use one of these methods:

Method 1 — Terminal (recommended)

Open Terminal and run:

xattr -cr ~/Downloads/ManifoldIndex.app

Then double-click the app normally.

Method 2 — System Settings

  • Double-click the app (it will be blocked)
  • Open System Settings → Privacy & Security
  • Scroll down — click "Open Anyway" next to the ManifoldIndex message
  • Click Open in the confirmation dialog

This is only needed once. Subsequent launches work normally.

Windows

Extract the zip and run ManifoldIndex.exe. Windows Defender may show a warning — this is normal for unsigned executables:

If blocked by Windows Defender

  • Click "More info" in the warning dialog
  • Click "Run anyway"

This is only needed on first launch. The app will run normally afterward.

Requirements

macOS

  • macOS 11 (Big Sur) or later
  • Apple Silicon Mac (M1 / M2 / M3 / M4)
  • No Python installation needed — everything is bundled

Windows

  • Windows 10 or later (Windows 11 recommended)
  • x64 processor
  • No Python installation needed — everything is bundled

Getting Started

  • Download and unzip ManifoldIndex.zip
  • Move ManifoldIndex.app to Applications (or anywhere you like)
  • Clear the quarantine flag (see Installation above)
  • Launch the app — enter a manifold name (e.g. m004) and click Run

Build from Source

Requires Python ≥ 3.10 and the packages listed in pyproject.toml.

  • pip install -e ".[dev]"
  • manifold-index to launch the GUI
  • pytest tests/ to run the test suite

Changelog

v0.5.0

  • Performance: Yield CPU and release GIL in progress callbacks for improved macOS compatibility and responsiveness
  • Error handling: Comprehensive error analysis and fixes in data handling and worker management
  • Stability: Enhanced session management with stale signal guards in export and filling cards
  • Defensive coding: Added null checks in filling card formatter calls
  • 76+ tests pass

v0.4.2

  • Critical fix: half-integer e summation in Dehn filling — valid_half_integer_patterns now enumerates all δ patterns instead of pre-filtering by internal columns only. The previous filter missed δ patterns that become valid when cusp charges (m, e) introduce cancelling half-integer contributions via g⁻¹. Affected most manifolds with odd SnapPy longitude coefficients (m003, m006, m007, m009, m015, m125, …).
  • Multi-cusp Weyl symmetry fix: ABVectors extended with per-cusp cusp_columns, shift_x2() method, and per-cusp accessors. Fixes garbage Weyl vectors for multi-cusp manifolds (e.g. m125).
  • Weyl convention text corrected: f(m,e) = f(−m,−e) (boundary charge flip), not f(η) = f(η⁻¹).
  • Removed stale Iref data pack (computed before the half-integer fix; will be recomputed).
  • 76 tests pass.

v0.4.0

  • Complete rewrite of computation core with cleaner architecture
  • IS chain intermediate steps use full (½)ℤ² lattice — fixes missing odd-m entries for all ℓ≥3 slopes
  • Pre-computed kernels-qq50 data pack fully recomputed (75 slopes, all correct)
  • Improved kernel builder: explicit pause/resume via bin/ scripts
  • All tests passing

v0.3.9

  • Window title and application name version strings now consistent
